서비스 소개 & 로그인

예상 시간: 1~2분
# 행동 설명 포인트
① 메인 페이지 열기 서비스 소개 화면, 학교 검색 기능 간단 소개. "교사·학생·학부모·관리자가 하나의 플랫폼에서 연결됩니다."
② 소셜 로그인 화면 보여주기 로그인/회원가입 흐름이 이렇게 생겼다는 것만 보여주고 패스. 실제 인증은 미리 준비된 계정 사용.
③ 교사 계정으로 로그인 후 허브 화면 역할 선택 허브(Hub) 화면 — "로그인하면 본인의 역할에 맞는 대시보드로 이동합니다." → 교사 선택.
1

교사 시점 — 정보를 만들고 전달한다

예상 시간: 5~7분 | 가장 핵심 파트
# 행동 설명 포인트
① 교사 대시보드 확인 담임 학급 학생 목록, 공지사항, 할 일(TODO) 위젯. "교사가 오늘 처리할 일들을 한눈에 파악합니다."
② 출결 관리 → 오늘 출결 입력 학생 목록에서 출석·지각·결석 체크. "교사가 입력하면 학생과 학부모 화면에 즉시 반영됩니다." (후반부에 학생/학부모 탭에서 확인)
③ 과제 생성 제목, 마감일, 첨부파일 포함해서 과제 하나 생성. "방금 생성한 과제가 학생에게 바로 보입니다."
④ 알림장 작성 (에디터) React Quill 리치 에디터로 알림장 작성 및 등록. "학부모에게도 바로 공유됩니다."
⑤ 상담 예약 현황 확인 선택 학부모가 신청한 상담 예약 목록. 시간 여유 있을 때만.
"방금 교사가 입력한 출결, 과제, 알림장이 학생 화면에서 어떻게 보이는지 확인해보겠습니다." → 학생 탭으로 전환
2

학생 시점 — 대시보드에서 모든 정보를 한눈에

예상 시간: 4~5분
# 행동 설명 포인트
① 학생 대시보드 확인 오늘 시간표, 급식, 알림장, 학급 게시판 등이 위젯으로 배치됨. "로그인하면 오늘 필요한 정보가 모두 여기 있습니다."
② 알림장 확인 교사가 방금 작성한 알림장이 대시보드 위젯 및 알림장 메뉴에 바로 표시됨.
③ 과제 목록 → 과제 제출 교사가 생성한 과제가 목록에 나타남. 파일 첨부 후 제출. "제출 현황은 교사와 학부모도 확인할 수 있습니다."
④ 성적 / 시험 일정 조회 시험 일정 캘린더, 과목별 성적 확인.
⑤ 퀴즈 풀기 선택 퀴즈 목록에서 하나 선택해 1~2문제 풀기. 시간 여유 있을 때만.
"이번엔 학부모 입장에서 자녀 상황을 어떻게 파악하는지 보겠습니다." → 학부모 탭으로 전환
3

학부모 시점 — 자녀 현황을 언제 어디서든

예상 시간: 3~4분
# 행동 설명 포인트
① 학부모 대시보드 확인 자녀 프로필 카드, 자녀 정보 요약. 자녀가 여러 명이면 탭으로 전환 가능.
② 자녀 출결 현황 확인 교사가 입력한 오늘 출결 상태가 여기 반영됨. "교사가 체크한 순간 학부모도 확인할 수 있습니다."
③ 자녀 과제 조회 학생이 제출한 과제 현황을 학부모도 확인. "자녀가 과제를 제출했는지 학부모가 직접 챙길 수 있습니다."
④ 상담 예약 신청 캘린더에서 날짜·시간대 선택 후 상담 신청. "교사와 학부모가 앱 안에서 상담 일정을 조율합니다."
"마지막으로 학교 전체를 관리하는 관리자 화면을 보겠습니다." → 관리자 탭으로 전환
4

관리자 시점 — 학교 전체를 한 화면에서 관리

예상 시간: 3~4분
# 행동 설명 포인트
① 관리자 대시보드 전체 학생 수, 교사 수, 승인 대기 현황, 학급별 통계 차트. "학교 전체 현황을 수치로 파악합니다."
② 학생 관리 목록 검색·필터링·정렬 기능. 학생 상세 정보 클릭해서 보여주기.
③ 학급 관리 학급 목록, 담임 배정 현황. "학급 생성부터 담임 배정까지 관리자가 처리합니다."
④ 공지사항 작성 선택 에디터로 공지사항 작성. 시간 여유 있을 때만.
⑤ 접속 로그 / 변경 이력 선택 감사(Audit) 기능. "누가 언제 어떤 데이터를 변경했는지 추적합니다." 시간 여유 있을 때만.

Q&A 준비

Q. 실시간 알림은 어떻게 구현했나요?
WebSocket(Socket.js) 기반으로 서버에서 이벤트를 푸시합니다. 새 과제, 공지사항, 상담 예약 확정 등을 상단 알림 드롭다운에서 실시간으로 받습니다.
Q. 역할 권한 관리는 어떻게 되나요?
STUDENT / TEACHER / PARENT / STAFF / ADMIN 역할별로 접근 가능한 라우트가 분리되어 있으며, 관리자는 세부 권한(학생관리자, 시설관리자 등)을 위임받을 수 있습니다.
Q. 한 사람이 여러 역할을 가질 수 있나요?
네, 허브(Hub) 화면에서 역할을 선택해 전환할 수 있습니다. 예를 들어 교사이면서 학부모인 경우도 지원합니다.
Q. PWA 지원이 있나요?
Service Worker와 Web App Manifest를 통해 모바일에서 홈 화면에 설치해 앱처럼 사용할 수 있습니다. 메인 페이지 접속 시 설치 안내가 표시됩니다.
Q. 기술 스택이 무엇인가요?
프론트엔드: React 19 + TypeScript + Vite, UI: Tailwind CSS + Radix UI, 라우팅: React Router v7, 차트: Recharts, 에디터: React Quill, 상태관리: Context API.
